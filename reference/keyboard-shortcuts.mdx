## Overview

The git-cx TUI (Terminal User Interface) provides an interactive workflow for creating commit messages. Different keyboard shortcuts are available depending on which screen you're currently on.

---

## Keyboard Shortcuts by Screen

### Select Type Screen

When selecting the commit type (feat, fix, docs, etc.):

| Key | Action |
|-----|--------|
| `↑` | Move selection up |
| `↓` | Move selection down |
| `Enter` | Confirm selected type and proceed |

Available commit types include:
- `feat` - A new feature
- `fix` - A bug fix
- `docs` - Documentation changes
- `style` - Code style changes (formatting, etc.)
- `refactor` - Code refactoring
- `perf` - Performance improvements
- `test` - Adding or updating tests
- `build` - Build system changes
- `ci` - CI/CD changes
- `chore` - Other changes that don't modify src or test files
- `revert` - Revert a previous commit

---

### Select Message Screen

When selecting from AI-generated commit message candidates:

| Key | Action |
|-----|--------|
| `↑` | Move selection up |
| `↓` | Move selection down |
| `Enter` | Confirm selected message and proceed |

<Note>
This screen displays the commit message candidates generated by your configured AI provider. The number of candidates shown depends on your `cx.candidates` configuration (default: 3).
</Note>

---

### Input Scope Screen

When entering the commit scope (optional):

| Key | Action |
|-----|--------|
| `Enter` | Accept entered scope and proceed to next screen |
| `Backspace` | Delete previous character |
| `Ctrl+W` | Delete previous word |
| `Ctrl+U` | Clear entire input |

<Note>
If you have configured `cx.commit.scopes`, the TUI will suggest these scopes as you type. Scopes help organize commits by area of the codebase (e.g., api, ui, core).
</Note>

**Examples of good scopes:**
- `api` - Changes to API endpoints
- `ui` - User interface changes
- `auth` - Authentication-related changes
- `core` - Core functionality changes
- `deps` - Dependency updates

---

### Input Body Screen

When entering the commit body (optional, multi-line):

| Key | Action |
|-----|--------|
| `Ctrl+D` | Finish entering body and proceed |
| `Enter` (on empty line) | Skip body and proceed |
| `Enter` (on non-empty line) | Add new line |
| `Backspace` | Delete previous character |
| `Ctrl+W` | Delete previous word |
| `Ctrl+U` | Clear current line |

<Note>
The commit body is optional and provides additional context about the changes. It's useful for explaining the "why" behind the changes or providing implementation details.
</Note>

**Body best practices:**
- Wrap lines at 72 characters for better readability
- Separate paragraphs with blank lines
- Use bullet points (`-` or `*`) for lists
- Reference issue numbers (e.g., "Fixes #123")

---

### Input Footer Screen

When entering the commit footer (optional):

| Key | Action |
|-----|--------|
| `Enter` | Accept entered footer and proceed to confirmation |
| `Backspace` | Delete previous character |
| `Ctrl+W` | Delete previous word |
| `Ctrl+U` | Clear entire input |

<Note>
The footer is used for metadata like issue references, breaking changes, or other structured information.
</Note>

**Common footer patterns:**
- `Fixes #123` - Link to resolved issue
- `BREAKING CHANGE: description` - Indicate breaking changes
- `Reviewed-by: Name` - Add reviewer information
- `Refs #456` - Reference related issue

---

### Confirm Screen

When reviewing the final commit message before committing:

| Key | Action |
|-----|--------|
| `y` | Commit the changes with the composed message |
| `n` | Abort and do not commit |
| `q` | Abort and do not commit (same as `n`) |

<Warning>
Once you press `y`, the commit will be created immediately. Make sure to review the complete message carefully before confirming.
</Warning>

The confirmation screen displays your complete commit message in the Conventional Commits format:

```
type(scope): subject

body

footer
```

---

## Universal Shortcuts

These shortcuts work across all screens:

| Key | Action |
|-----|--------|
| `Ctrl+C` | Force quit git-cx (emergency exit) |
| `Esc` | Return to previous screen (where applicable) |

<Note>
Using `Ctrl+C` will immediately terminate git-cx without creating a commit, regardless of which screen you're on.
</Note>

---

## TUI Navigation Flow

The TUI guides you through the following screens in order:

1. **Select Type** → Choose commit type (feat, fix, docs, etc.)
2. **Select Message** → Pick AI-generated subject line
3. **Input Scope** → Enter optional scope
4. **Input Body** → Enter optional body (multi-line)
5. **Input Footer** → Enter optional footer
6. **Confirm** → Review and confirm the final commit message

You can use `Enter` to proceed through each step, and press `Enter` on an empty input to skip optional fields.

---

## Dry Run Mode

When using `--dry-run`, the TUI behaves the same way but:
- The final confirmation step is skipped
- No actual git commit is created
- The composed message is displayed for preview only

```bash
git cx --dry-run
```

This is useful for:
- Testing your configuration
- Previewing AI-generated messages without committing
- Experimenting with different providers or models

## Examples

### Quick Commit Workflow

```bash
# Stage your changes
git add -A

# Start git-cx
git cx

# Navigate through TUI:
# 1. Press ↓ to select "feat", press Enter
# 2. Press ↓ to review messages, press Enter on your choice
# 3. Type "api", press Enter
# 4. Press Enter (skip body)
# 5. Press Enter (skip footer)
# 6. Review and press 'y' to commit
```

### Detailed Commit with Body

```bash
git cx

# After selecting type and message:
# 1. Enter scope: "auth"
# 2. Enter body:
#    "Implement JWT token refresh mechanism
#    
#    This adds automatic token refresh when the access
#    token expires, improving user experience by
#    reducing forced logouts."
#    Press Ctrl+D
# 3. Enter footer: "Fixes #123"
# 4. Confirm with 'y'
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Arrow keys not working">
    Make sure your terminal emulator supports ANSI escape sequences. Most modern terminals (iTerm2, Terminal.app, Alacritty, Windows Terminal) work fine. If you're using an older terminal, try updating it.
  </Accordion>
  
  <Accordion title="TUI appears corrupted or garbled">
    This can happen if your terminal size is too small. Try maximizing your terminal window or increasing the font size. git-cx works best with terminals at least 80 columns wide and 24 rows tall.
  </Accordion>
  
  <Accordion title="Ctrl+D not working in body input">
    Some terminals may intercept `Ctrl+D`. Try pressing it twice, or use `Enter` on an empty line to skip the body field instead.
  </Accordion>
  
  <Accordion title="Can't abort from confirmation screen">
    Press `n`, `q`, or `Ctrl+C` to abort without committing.
  </Accordion>
</AccordionGroup>
